wsdl.exe -
    Программа для создания кода для клиентов XML-веб-службы и XML-веб-служб,
    использующих ASP.NET из файлов контракта WSDL, схемы XSD и обнаруженные
    документы discomap. Это средство можно использовать совместно с программой disco.exe.

wsdl.exe <параметры> <URL-адрес или путь> <URL-адрес или путь> ...

     - ПАРАМЕТРЫ -

<URL-адрес или путь> -
    URL-адрес или путь к контракту WSDL, схеме XSD или документу discomap.

/nologo
    Отмена вывода заголовка.

/language:<язык>
    Язык, используемый для создаваемого кода. Выберите "CS", "VB", "JS",
    "VJS", "CPP" или укажите полное имя для класса, реализующего
    System.CodeDom.Compiler.CodeDomProvider. Язык по умолчанию:
    "CS" (CSharp). Краткая форма: "/l:".

/sharetypes
    Включение совместного использования типов. Эта функция создает файл кода с
    одним определением типа для идентичных типов, которое используется разными
    службами (пространство имен, имя и сетевая сигнатура должны быть идентичными).
    Ссылайтесь на службы с помощью URL-адресов http:// в параметрах командной строки
    или создайте документ discomap для локальных файлов.

/verbose
    Отображение дополнительных сведений, если указан переключатель /sharetypes.
    Краткая форма: "/v".

/fields
    Создание полей вместо свойств. Краткая форма: "/f".

/order
    Создание явных идентификаторов последовательности в членах примитива.

/enableDataBinding
    Реализация интерфейса INotifyPropertyChanged во всех создаваемых типах
    для обеспечения привязки данных. Краткая форма: "/edb".

/namespace:<пространство_имен>
    Пространство имен для созданного прокси-класса или шаблона. По умолчанию используется
    глобальное пространство имен. Краткая форма: "/n:".

/out:<имя_файла|путь_к_каталогу>
    Имя файла или путь к каталогу для созданного кода прокси. По умолчанию
    имя файла наследует имя класса. Краткая форма: "/o:".

/protocol:<протокол>
    Изменение протокола, реализуемого по умолчанию. Можно выбрать "SOAP",
    "SOAP12", "HttpGet", "HttpPost".

/username:<имя_пользователя>
/password:<пароль>
/domain:<домен>
    Учетные данные, используемые для подключения к серверу,
    требующему аутентификацию. Краткие формы: "/u:", "/p:" и "/d:".

/proxy:<URL-адрес>
    URL-адрес прокси-сервера, используемого для HTTP-запросов.
    По умолчанию используются системные параметры прокси-сервера.

/proxyusername:<имя_пользователя>
/proxypassword:<пароль>
/proxydomain:<домен>
    Учетные данные, используемые для подключения к прокси-серверу,
    требующему аутентификацию. Краткие формы: "/pu:", "/pp:" и "/pd:".

/appsettingurlkey:<ключ>
    Ключ конфигурации, используемый во время создания кода для чтения значения
    по умолчанию свойства URL. По умолчанию чтение из файла конфигурации
    не производится. Краткая форма: "/urlkey:".

/appsettingbaseurl:<базовый URL-адрес>
    Базовый URL-адрес, используемый для вычисления фрагмента URL-адреса. Также
    необходимо указать параметр appsettingurlkey. Фрагмент URL-адреса -
    это результат преобразования относительного URL-адреса из параметра appsettingbaseurl
    в URL-адрес в документе WSDL. Краткая форма: "/baseurl:".

/parsableerrors
                 Печать ошибок в формате, похожем на формат, используемый компиляторами.

     - РАСШИРЕННАЯ -

/server
    Не рекомендуется использовать переключатель сервера. Вместо него используйте переключатель /serverInterface.
    Создание абстрактного класса для реализации XML-веб-службы с помощью
    ASP.NET на основе контрактов. По умолчанию создаются клиентские
    прокси-классы.

/serverInterface
    Создание интерфейсов для реализации веб-службы ASP.Net 
    на стороне сервера. Для каждой привязки в документах WSDL создается 
    интерфейс. Сам по себе WSDL реализует контракт WSDL (классы, 
    реализующие интерфейс, не  должны включать следующие
    методы класса: атрибуты веб-службы или атрибуты 
    сериализации, изменяющие контракт WSDL). Краткая форма: "/si".

/parameters:<файл>
    Чтение параметров командной строки из указанного xml-файла. Это позволяет
    указать параметры, недоступные из командной строки, например выбрать
    тип создаваемой модели асинхронного программирования. Для получения дополнительных сведений
    см. документацию к средству. Краткая форма: "/par:".
